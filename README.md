# ConsoleBanking
(RU Version)
### Classes
1. UserProfile - Хранит всю информацию о пользователе и его профиле
2. Interface - Интерфейс и взаимодействие с интерфейсом банка
3. Auth - Создание, хранение, доступ к профилям
4. Transactions - Логика транзакций
5. History - Истории транзакций
6. ConsoleBanking - Основной класс

### Logic of work and interaction between classes
- Запуск приложения происходит через основной класс ConsoleBanking. После страта основного кода первым запускается класс Interface выводя страницу авторизации. Независимо от того создается новый аккаунт или происходит вход в существующий, логин и пароль отправляются на обработку в класс Auth, где с этими данными будет произведен вход в аккаунт или его создание, либо выдана ошибка о неверно введенном логине или пароле. При успешном входе/регистрации или ошибке входа, будет выведен соответствующий экран, главного меню банка или ошибки при входе.  Вывод любого экрана/интерфейса производится классом  Interface.
    - При успешном входе в аккаунт класс Auth возвращает объект класса UserProfile информацией из которого будет заполнено главное меню выведенное классом Interface.
    - Транзакции обрабатываются классом Transactions, он же поверхностно взаимодействует с классом Auth для проверки существования профиля.
    - Класс History принимает информацию о каждой успешной транзакции и сохраняет ее. Так же он предоставляет возможность классу Interface получить доступ к истории и вывести ее в меню банка.
